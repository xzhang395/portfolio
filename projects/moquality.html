INTRO

During my 3-month's contract work at the startup company MoQuality. I was challenged to design a platform for automated mobile app testing from scratch. As the only designer on the team, I worked closely with the developers/executives to understand the technology and company goal. Iterate fast, test the designs with potential users and use insights to guide product direction.


GETTING STARTED

How to help user understand testing results?

MoQuality's initial product was a program that randomly clicks on any buttons on the screen and detect possible usability issues. An "App Map"is then generated to display all the screens with issues highlighted. I started by designing the dashboard for improving the usability of the app map.

The original version of the App Map displays all screens tested by the program and highlights the errors.


The design focused on helping user quickly navigate through all the issues and understand the issues by having visual and verbal explanations side by side.


Yet after testing with the potential users, having a machine randomly clicking on the app turns out to be an inefficient way of testing, because the users only care about how a real person would use the app and any errors that happens along the way.

“I don't care about the meaningless map randomly generated, I only care about if any issue will happen when a real user is using my app.

Let user create the flow

NEEDS

What's missing from current testing methods?

There is a long list of challenges for mobile app testing. Instead of solving all of them at the same time, we want to target the biggest issues faced by the users. After interviewing couple mobile app development startups, the biggest challenge is access to the right set of devices when there is an ever-growing list of devices and operating system versions.


The common practice in small mobile app companies is to buy a set of the most common devices on the market and test separately. Yet the problems are:

  • Buying a full set of devices is expensive and even impossible
  • Repeat testing on multiple devices is time consuming
  • Errors can easily be ignored by human testers

LIMITATIONS

Designing for the technology limitation

Ideally our product can test on all devices and present users with all the possible issues, but the fact is that our current technology is far from flawless. The machine learning algorithm needs more labeled data from human to be able to discover more complicated issues.
